# Дарование — CRM детского центра

Веб-система для учёта учеников, расписания, абонементов, оплат и отчётности детского центра развития талантов **«Дарование»**.

Стек: **Django 6**, **MySQL**, HTML/CSS/JS (без отдельного фронтенд-фреймворка).

## Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Kristina5946/DAROVANIE_sys_upravleny.git
cd DAROVANIE_sys_upravleny

# 2. Виртуальное окружение
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate

# 3. Зависимости
pip install -r requirements.txt

# 4. Настройки
copy .env.example .env        # Windows
# cp .env.example .env        # Linux/macOS
# Отредактируйте .env — пароль MySQL, SECRET_KEY

# 5. Создать БД в MySQL (через phpMyAdmin или консоль)
# CREATE DATABASE darovanie_crm CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 6. Миграции и суперпользователь
python manage.py migrate
python manage.py createsuperuser

# 7. (Опционально) Демо-данные
python manage.py seed_demo_data

# 8. Запуск
python manage.py runserver 8080
```

Откройте в браузере: **http://127.0.0.1:8080/**

## Настройка окружения

Файл `.env` (создаётся из `.env.example`):

| Переменная | Описание | Пример |
|------------|----------|--------|
| `DJANGO_SECRET_KEY` | Секретный ключ Django | длинная случайная строка |
| `DJANGO_DEBUG` | Режим отладки | `True` / `False` |
| `DJANGO_ALLOWED_HOSTS` | Разрешённые хосты | `localhost,127.0.0.1` |
| `DB_NAME` | Имя базы | `darovanie_crm` |
| `DB_USER` | Пользователь MySQL | `root` |
| `DB_PASSWORD` | Пароль MySQL | ваш пароль |
| `DB_HOST` | Хост | `127.0.0.1` |
| `DB_PORT` | Порт | `3306` |

> **Важно:** файл `.env` не попадает в git. Никогда не коммитьте реальные пароли.

## Первый запуск

### 1. Создание директора

```bash
python manage.py createsuperuser
```

Укажите логин, email (можно пустой) и пароль.

После создания пользователя автоматически создаётся профиль с ролью. Суперпользователь получает полный доступ.

### 2. Назначение роли «Директор»

1. Войдите в **Настройки** (`/admin/`) под суперпользователем.
2. Откройте **Пользователи** → нужный пользователь.
3. В блоке **«Доступ в CRM»** выберите роль **Директор**.
4. Сохраните.

Роль применяется автоматически: директору выставляется `is_staff` и открывается доступ к финансам и admin.

### 3. Демо-данные

```bash
python manage.py seed_demo_data          # добавить демо-записи
python manage.py seed_demo_data --clear  # очистить и заполнить заново
```

Создаёт: 10 направлений, учеников, преподавателей, расписание, оплаты, абонементы и т.д.  
Для оплат нужен существующий суперпользователь — создайте его до запуска команды.

## Роли и доступ

Система ролей реализована в приложении `accounts` (модель `UserProfile`).

| Роль | Меню и функции |
|------|----------------|
| **Директор** | Всё: оплаты со сводками, отчёты, графики, Excel, Django Admin |
| **Ресепшн** | Ученики, преподаватели, абонементы, направления, расписание, **оплаты без итогов и графиков** |

### Что видит ресепшн в разделе «Оплаты»

- список оплат, фильтры, добавление и редактирование записей;
- **без** блока «Всего / Записей / Средняя»;
- **без** строки «Итого» внизу таблицы;
- **без** графиков и экспорта в Excel.

### Что недоступно ресепшну

- раздел **Отчёты**;
- пункт **Настройки** (Django Admin);
- финансовые виджеты на главной странице.

### Как добавить сотрудника ресепшн

1. Директор → **Настройки** → **Пользователи** → **Добавить**.
2. Логин, имя, **пароль** (обязательно при создании).
3. Роль в CRM: **Ресепшн**.
4. **Не включайте** вручную «Статус персонала» и «Суперпользователь» — права выставятся из роли.
5. Сохранить.

### Безопасность

- У каждого сотрудника **свой** логин и пароль.
- Ресепшн не может зайти в `/admin/` (`is_staff = False`).
- Финансовые URL защищены декоратором `@admin_required` (проверка роли директора).
- Пароли хранятся в хэшированном виде (стандарт Django).

## Разделы системы

### Главная (`/`)

- Быстрые кнопки: посещаемость сегодня, новый ученик, оплаты.
- Глобальный поиск по ученикам.
- Справка с подсказками по работе (раскрывающиеся блоки).
- У директора: метрики месяца и столбчатый график доходов за 6 месяцев.

### Ученики (`/students/`)

**Список** — поиск, фильтр по направлению.

**Карточка ученика:**

- профиль (ФИО, родитель, телефон, направления);
- блок **«Направления и абонементы»** — оформление абонемента, остаток занятий;
- вкладка **Оплаты** — просмотр, добавление, редактирование (суммы предзаполнены);
- вкладка **Посещения** — история, отметки присутствия и оплаты занятия.

### Преподаватели (`/teachers/`)

- Привязка к направлениям.
- Вкладки: ученики по направлениям, посещения, **начисления** (расчёт зарплаты за период).

### Абонементы (`/subscriptions/`)

Сетка карточек активных абонементов с датами окончания и фильтрами.

### Оплаты (`/reports/payments/`)

- Фильтры: период, направление, тип, поиск по ученику.
- Режим редактирования: массовое изменение даты, суммы, типа, направления.
- Добавление новой оплаты через форму «+ Добавить оплату».

### Направления (`/directions/`)

Типы занятий:

- **Групповое** (`group`) — несколько учеников на одном слоте расписания.
- **Индивидуальное** (`individual`) — в расписании указывается конкретный ученик.

Цены на карточке направления:

| Поле | Использование |
|------|---------------|
| `price_per_lesson` | Стоимость занятия по абонементу |
| `single_lesson_cost` | Разовое / пробное занятие |
| `subscription_cost` | Рекомендуемая цена абонемента |

### Расписание (`/schedule/`)

Режимы просмотра:

- **Сегодня** — отметка посещаемости на текущий день;
- **Неделя** — сетка по дням;
- **Таблица** — редактирование слотов.

Дополнительно:

- разовые занятия и отработки;
- отмена занятия (исключение в расписании);
- для индивидуальных направлений — выбор ученика в слоте.

### Отчёты (`/reports/`) — только директор

- Сводный дашборд за месяц.
- Отчёт по зарплатам преподавателей.
- Отчёт по прибыли.
- Ссылки на детализацию оплат.

## Типовые сценарии работы

### Новый ученик

1. **Ученики** → **Добавить** или кнопка на главной.
2. Заполнить ФИО, родителя, телефон, выбрать направления.
3. На карточке — **+ Направление** (если нужно добавить позже).
4. Оформить абонемент в блоке направления или внести оплату на вкладке **Оплаты**.

### Отметить посещение

1. **Расписание** → режим **Сегодня** (или выбрать дату).
2. Отметить галочками присутствие и оплату занятия.
3. Нажать **Сохранить**.

### Внести оплату

**Из карточки ученика:** вкладка **Оплаты** → **+ Добавить оплату**.

**Из общего журнала:** **Оплаты** → **+ Добавить оплату** → выбрать ученика.

### Оформить абонемент

На карточке ученика в блоке направления:

1. Указать количество занятий и сумму (подставляются автоматически из цен направления).
2. Нажать **Оформить абонемент**.

Создаётся запись абонемента и связанная оплата.

### Редактировать оплаты

- В карточке ученика: **Оплаты** → **✎ Редактировать**.
- В общем журнале: **Оплаты** → **✎ Редактировать**.

Все поля предзаполнены текущими значениями (дата, сумма, тип, примечание).

## Зарплата преподавателей

Расчёт упрощённый, без ставок и бонусов:

```
Начисление = сумма посещений × 30%
```

**Сумма посещения:**

- по **абонементу** — `price_per_lesson` направления;
- **разовое / пробное** — `single_lesson_cost` (или цена из разового занятия).

Учитываются только отмеченные посещения (`present = True`) за выбранный период.

Подробная разбивка — на странице преподавателя и в отчёте **Зарплаты**.

## Развёртывание на сервере

Краткий чеклист для production:

1. `DJANGO_DEBUG=False`
2. Надёжный `DJANGO_SECRET_KEY`
3. `DJANGO_ALLOWED_HOSTS` — ваш домен
4. `python manage.py migrate`
5. `python manage.py collectstatic`
6. Запуск через **gunicorn** / **uwsgi** + nginx
7. HTTPS (Let's Encrypt)

Пример переменных для production в `.env`:

```env
DJANGO_DEBUG=False
DJANGO_SECRET_KEY=сгенерируйте-длинный-случайный-ключ
DJANGO_ALLOWED_HOSTS=crm.example.ru,www.crm.example.ru
```

## Устранение неполадок

### Не подключается MySQL

- Проверьте, что MySQL запущен.
- Убедитесь, что база `darovanie_crm` создана.
- Сверьте `DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT` в `.env`.

### Пустые суммы в полях редактирования

Поля `type="number"` требуют точку как разделитель (`3500.00`, не `3500,00`).  
В проекте используется фильтр `input_number` и `localize=False` в формах — при кастомных шаблонах соблюдайте это правило.

### Ресепшн видит отчёты или admin

Проверьте роль в **Настройки → Пользователи → Доступ в CRM**.  
Должно быть **Ресепшн**, не Директор.

### После `git pull` ошибки моделей

```bash
pip install -r requirements.txt
python manage.py migrate
```
